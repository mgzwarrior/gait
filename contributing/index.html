Contribution Guidelines

Design Principles

  1. Simplicity - The code should be simple and easy to understand. This is the most important principle. If you are not sure if your code is simple enough, ask for a review.
  2. KISS - Keep It Simple, Stupid
  3. Consistency - The code should be consistent with the rest of the codebase. This means that it should follow the same coding style, use the same terminology, and use the same patterns as the rest of the codebase.
  4. Completeness - The code should be complete. This means that it should handle all possible cases, and should not leave any edge cases unhandled.
  5. Correctness - The code should be correct. This means that it should produce the correct output for all possible inputs.
  6. Efficiency - The code should be efficient. This means that it should produce the correct output in the least amount of time and space possible.
  7. Robustness - The code should be robust. This means that it should handle all possible errors and exceptions gracefully.
  8. Testability - The code should be testable. This means that it should be easy to write tests for the code.
  9. Extensibility - The code should be extensible. This means that it should be easy to add new features to the code.
  10. Modularity - The code should be modular. This means that it should be easy to reuse the code in other projects.
  11. Maintainability - The code should be maintainable. This means that it should be easy to maintain the code in the long run.
  12. Portability - The code should be portable. This means that it should work on all platforms and environments.
  13. Security - The code should be secure. This means that it should not contain any security vulnerabilities.
  14. Documentation - The code should be documented. This means that it should be easy to understand the code without reading the code itself.
  15. Readability - The code should be readable. This means that it should be easy to read and understand the code.
  16. Scalability - The code should be scalable. This means that it should be easy to scale the code to handle large amounts of data.
